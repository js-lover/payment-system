(kullanılmıyor)\n- `WaitingForApproval` (1) - Yeni kart, onay bekliyor\n- `Active` (2) - Aktif, kullanılabilir\n- `Blocked` (3) - Bloklanmış\n- `Expired` (4) - Süresi dolmuş\n\n**S: CVC'yi nasıl şifrelemeliyim?**\n\nC: CARD_CODE_EXAMPLES.md'deki `EncryptCVC()` metoduna bakın. Gerçek production'da AES encryption kullanın.\n\n**S: Kart silinirse Transaction'lar ne olur?**\n\nC: Foreign key 'OnDelete: Cascade' ayarlı olduğu için kartla ilgili transactions da silinir. Soft delete istiyorsanız status flag ekleyin.\n\n**S: Bir kullanıcı başka birinin kartını görebilir mi?**\n\nC: Hayır. Authorization kontrolü yapılıyor. Sadece kendi Account'ının kartlarını görebilir.\n\n---\n\n## 📞 Destek & Referanslar\n\n- **Architecture:** Clean Architecture / Layered Architecture\n- **Pattern:** Repository Pattern + Service Pattern\n- **ORM:** Entity Framework Core 10\n- **Framework:** .NET 10.0 / ASP.NET Core\n- **Database:** SQLite\n\n---\n\n## ✅ Başarı Kontrol Listesi\n\nİmplementasyon tamamlandığında bu kontrol listesini tıkla:\n\n- [ ] Repository implementation tamamlandı\n- [ ] Service implementation tamamlandı\n- [ ] DTOs tamamlandı\n- [ ] AutoMapper profile tamamlandı\n- [ ] Controller endpoints tamamlandı\n- [ ] DI registration tamamlandı\n- [ ] Database migration çalıştırıldı\n- [ ] Unit tests yazıldı\n- [ ] Integration tests yazıldı\n- [ ] Manual tests başarılı\n- [ ] Swagger documentation eklendi\n- [ ] Code review tamamlandı\n- [ ] Production'a hazır ✅\n\n---\n\n## 📌 Sonraki Adımlar\n\n1.
